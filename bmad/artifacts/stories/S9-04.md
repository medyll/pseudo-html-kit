# Story S9-04 — Notification: autodismiss animation + Interest Invokers

**Epic:** Migration
**Sprint:** Sprint 09
**Points:** 3
**Priority:** Must

## User Story

As a **vanilla JS developer**, I want notifications to auto-dismiss after a configurable duration using CSS animation, and pause on hover/focus, so that I don't need JavaScript timers for this behavior.

## Context

Current notification is static — it renders and stays until manually removed. This story adds an `[autodismiss]` attribute that triggers a CSS fade-out animation. Hover/focus pauses the animation via `:hover`/`:focus-within` (all browsers) and `:interest` (Chrome 146+). See tech-spec section 4 (Notification).

Traces: MIG-09, MIG-10

## Acceptance Criteria

```gherkin
Given a notification with [autodismiss] attribute
When it renders
Then it fades out after 5s via CSS animation

Given a notification with [autodismiss] and --dismiss-duration: 3s
When it renders
Then it fades out after 3s

Given a notification is auto-dismissing
When the user hovers over it
Then the animation pauses

Given the user stops hovering
Then the animation resumes

Given a notification with [autodismiss]
When the animation ends
Then the element is removed from DOM

Given a notification with [dismissible] attribute
When the dismiss button is clicked
Then the element is removed from DOM immediately

Given a notification WITHOUT [autodismiss]
When it renders
Then it behaves exactly as before (static, no animation)

Given a browser supporting :interest pseudo-class
When the user gives interest to the notification (hover/focus proximity)
Then the animation pauses (enhanced path)
```

## Technical Notes

- **No HTML template changes** — all behavior via CSS + script
- CSS: add `@keyframes pk-notif-dismiss` (opacity 1->0, translateY)
- CSS: `:scope[autodismiss]` applies the animation
- CSS: `:scope[autodismiss]:hover, :scope[autodismiss]:focus-within` pauses animation
- CSS: `@supports selector(:interest)` block for `:interest` enhancement
- CSS custom property `--dismiss-duration` defaults to `5s`
- Script: add `animationend` listener to remove element from DOM when animation name matches
- Script: add click listener on `.notification__dismiss` button to `el.remove()`
- New prop: `autodismiss` (boolean attribute)
- New CSS custom property: `--dismiss-duration` (optional override)

## Out of Scope

- Toast/snackbar stacking (multiple notifications)
- Programmatic dismiss API
- Slide-in entrance animation
