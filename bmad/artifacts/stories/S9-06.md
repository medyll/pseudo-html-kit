# Story S9-06 — E2E tests for native API behavior (Playwright)

**Epic:** Testing
**Sprint:** Sprint 09
**Points:** 3
**Priority:** Should

## User Story

As a **developer**, I want E2E tests that validate native `<dialog>`, Popover, and Anchor Positioning behavior in real browsers, so that I'm confident the migration works beyond mocked unit tests.

## Context

Unit tests (S9-05) use mocks for new APIs. E2E tests run in real Chromium (146+) to validate actual browser behavior: focus trapping, light-dismiss, anchor positioning, CSS animation.

Traces: TST-03, TST-04

## Acceptance Criteria

```gherkin
Given a modal-pk in Chromium
When opened via trigger
Then focus is trapped within the dialog
And Escape key closes it
And backdrop click closes it

Given a dropdown-pk in Chromium
When trigger is clicked
Then menu appears as popover
And clicking outside dismisses it

Given a tooltip-pk in Chromium
When hovered near viewport edge
Then tooltip auto-flips position

Given a notification-pk with [autodismiss] in Chromium
When 5 seconds elapse
Then the notification fades out and is removed from DOM
When hovered during countdown
Then the animation pauses
```

## Technical Notes

- Create `tests/migration-e2e.e2e.js`
- Use existing Playwright config (`playwright.config.js`)
- Serve a test page with all 4 migrated components
- Tests run in Chromium (native API path)
- Optional: run subset in Firefox/WebKit to validate fallback (S9-07)

## Out of Scope

- Full cross-browser matrix (covered in S9-07)
- Performance benchmarking

## Dependencies

- S9-05 (unit tests pass first)

## Definition of Done

- [ ] `tests/migration-e2e.e2e.js` created
- [ ] Modal: focus trap + Escape + backdrop close tested
- [ ] Dropdown: popover toggle + light-dismiss tested
- [ ] Tooltip: positioning validated (at least top + bottom)
- [ ] Notification: autodismiss + hover-pause tested
- [ ] All tests pass in Chromium
