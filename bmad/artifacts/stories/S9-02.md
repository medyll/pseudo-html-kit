# Story S9-02 — Dropdown: migrate to Popover API + `popovertarget`

**Epic:** Migration
**Sprint:** Sprint 09
**Points:** 5
**Priority:** Must

## User Story

As a **vanilla JS developer**, I want the dropdown menu to use the native Popover API, so that I get light-dismiss (click outside), top-layer stacking, and declarative toggle without custom JS.

## Context

Current dropdown uses CSS `display: none` / `display: block` toggle via `[open]` attribute on host. No JS in the component. Migration adds `[popover]` to the menu and `popovertarget` to the trigger. See tech-spec section 3 (Dropdown).

Traces: MIG-07, MIG-08

## Acceptance Criteria

```gherkin
Given a dropdown-pk component is registered
When the trigger button is clicked
Then the menu shows via Popover API (popover-open state)

Given a dropdown menu is open
When the user clicks outside the dropdown
Then the menu closes via native light-dismiss

Given the dropdown opens
Then aria-expanded on trigger updates to "true"

Given the dropdown closes
Then aria-expanded on trigger updates to "false"

Given the dropdown opens
Then the host [open] attribute is set

Given the dropdown closes
Then the host [open] attribute is removed

Given a browser that does NOT support Popover API
When the trigger is clicked
Then a fallback JS click handler toggles the [open] attribute
And a click-outside listener closes the dropdown

Given dropdown has align="right"
Then the menu aligns to the right edge of the trigger

Given dropdown has disabled attribute
Then the trigger button is disabled and menu cannot open
```

## Technical Notes

- Add `popover` attribute + unique `id` to `<ul class="dropdown__menu">`
- Add `popovertarget` attribute on trigger button (set via script with generated uid)
- Script: generate unique ID (`dd-` + random) to avoid conflicts with multiple dropdowns
- Script: listen to `toggle` event on menu to sync `aria-expanded` and host `[open]`
- Script: feature detect `'popover' in HTMLElement.prototype` for fallback
- Fallback: remove `popover`/`popovertarget`, wire manual click toggle + click-outside
- CSS: use `@supports (anchor-name: --x)` for anchor-positioned menu; `@supports not` for absolute fallback
- CSS: `.dropdown__menu:popover-open` for arrow rotation (alongside existing `:scope[open]`)
- Preserve slots: trigger, items
- Preserve align="left|right" and disabled props

## Out of Scope

- Keyboard navigation within menu items (separate story)
- Nested/sub-dropdowns
