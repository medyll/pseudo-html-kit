## Implementation Notes

**Date:** 2026-03-03
**Files changed:**
- `tests/atoms.client.test.js` — new; 17 describe blocks, ~68 tests
- `vitest.config.js` — include glob updated to cover `atoms.client.test.js`

**Notable decisions:**
- Load actual HTML files with `fs.readFileSync` so tests stay in sync with component source.
- Focus on observable DOM output (class presence, slot wiring, resolved state) — no CSS assertion.
- Script blocks inside atom HTML are static (no `el.dataset.*` mutations), so script-execution tests are skipped for atoms.

**Known limitations:**
- CSS `@scope` rules are not exercised (happy-dom ignores them at parse time).
- Attribute-driven DOM mutations (e.g. progress bar width from `value`) that require inline script are not present in atoms — skip those assertions.
