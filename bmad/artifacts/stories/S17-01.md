## User Story
As a Svelte 5 developer, I want a `pseudo-kit-svelte` package that exposes async functions compatible with `$effect`, so I can load pseudo-html-kit components without coupling to React or a specific framework runtime.

## Implementation Notes

**Date:** 2026-03-11
**Files changed:**
- `src/pseudo-kit-svelte/src/index.js` — exports: `nameFromUrl`, `createComponent`, `createComponents`, `pseudoKit`, `initPseudoKit`
- `src/pseudo-kit-svelte/package.json` — v0.1.0, peer dep: svelte >=5
- `src/pseudo-kit-svelte/README.md` — usage examples with $state/$effect

**Notable decisions:**
- Exposed plain async functions (not Svelte runes) — rune wiring is the caller's responsibility; adapter stays compiler-agnostic
- `initPseudoKit(urls, root)` supports scoped init for Svelte component trees
- API mirrors pseudo-kit-react conceptually but without React-specific hooks
