## User Story

As a **vanilla JS developer**, I want the modal component to use the native `<dialog>` element, so that I get browser-managed focus trapping, Escape key handling, and `::backdrop` styling without custom JS.

## Context

Current modal uses `<div class="modal" role="dialog">` with manual CSS `display` toggle and a `.modal__backdrop` div. The `<dialog>` element provides all of this natively. See tech-spec section 1 (Modal).

## Implementation Summary

### Template Changes
- Replaced `<div class="modal" role="dialog" aria-modal="true">` with `<dialog class="modal">`
- Removed `.modal__backdrop` div â€” native `::backdrop` pseudo-element is used instead
- Preserved all 3 slots: `header`, `default`, `footer`
- Kept `aria-labelledby="modal-title"` for accessibility

### CSS Changes
- Removed fixed positioning and display toggle logic
- Let `<dialog>` handle default hidden state; `showModal()` shows it in the top layer
- `::backdrop` pseudo-element styled with semi-transparent overlay
- Preserved size variants: sm, md, lg, fullscreen
- Container centering now relies on `<dialog>`'s native centering

### Script Changes
- Added `MutationObserver` to detect `[open]` attribute changes
- Call `dialog.showModal()` when `[open]` is set; `dialog.close()` when removed
- Wire close button and backdrop click to `dialog.close()`
- Sync `dialog.close` event back to host by removing `[open]` attribute
- Handle title prop forwarding to `.modal__title` element

### Test Updates
- Updated `tests/molecules.client.test.js` modal test suite
- Changed test for ARIA attributes: `<dialog>` provides `role="dialog"` and `aria-modal="true"` implicitly
- Removed assertion for `.modal__backdrop` DOM node (no longer exists)
- All 51 client tests pass âś…

## Technical Notes

- `<dialog>` is supported in Chrome 118+, Firefox 128+, Safari 17.4+
- For older browsers, fallback behavior uses `showModal()` if available; silent no-op otherwise
- Focus trap and Escape key handling are provided by the browser
- `::backdrop` styling replaces the manual `.modal__backdrop` div
- External trigger button wiring with `popovertarget` is consumer responsibility (not in scope)
