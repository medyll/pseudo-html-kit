# Story S11-03 — Accordion: CSS @supports + View Transitions API

**Epic:** Layout Components Migration  
**Sprint:** Sprint 11  
**Points:** 3  
**Priority:** Should  

---

## User Story

As a **UI developer**, I want `accordion-pk` to animate open/close using the View Transitions API (`document.startViewTransition`), so that collapse/expand is smooth without JS animation libraries.

## Context

The View Transitions API (Chrome 111+) enables smooth DOM state transitions via `document.startViewTransition`. Combined with `<details>` / `<summary>` native semantics, this story creates an accessible accordion with native browser animation on modern browsers and a clean no-animation fallback.

## Acceptance Criteria

```gherkin
Given accordion-pk with multiple panels
When user clicks a summary/header
Then panel open/close animates smoothly via View Transitions (Chrome 111+)

Given accordion-pk on older browser
When View Transitions is not supported
Then panel toggles instantly without JS error

Given accordion-pk with exclusive attribute
When one panel is opened
Then other panels close (JS-managed exclusive mode)

Given accordion-pk
When navigated by keyboard
Then Tab/Enter/Space follow standard <details> semantics (accessible)
```

## Technical Notes

- New organism: `src/pseudo-assets/components/organisms/accordion-pk.html`
- Base: native `<details>` / `<summary>` for zero-JS toggle semantics
- View Transition: wrap `el.open = !el.open` with `document.startViewTransition?.()`
- CSS: `@keyframes` for slide-down/up; `::view-transition-*` for VT animation
- `exclusive` attribute: JS loop to close siblings before opening target
- `@supports (view-transition-name: none)` gate for VT-only styles
- Update `src/pseudo-assets/index.js` to export new organism

## Out of Scope

- Nested accordions
- Accordion groups linked to routing
- Animation customization API (duration/easing props)
