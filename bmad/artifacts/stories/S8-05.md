## Description

Fix all Critical and Serious accessibility violations surfaced by S8-04 audit. Target: 0 Critical + 0 Serious axe violations across all 46 components.

Violations from S8-04 initial analysis and resolution strategy:

| Rule | Severity | Fix |
|---|---|---|
| `label` | Critical | `input-pk` and `textarea-pk` scripts forward `aria-label`/`id` to inner field; fixture provides labels |
| `aria-progressbar-name` | Serious | `progress-bar-pk` script forwards `aria-label` to inner `role="progressbar"` |
| `aria-prohibited-attr` | Serious | Component scripts remove `aria-*` from outer custom elements and apply them to inner semantic elements |
| `color-contrast` | Serious | Fixture uses high-contrast token values (`#1d4ed8`, `#1a202c`); `.component-label` uses `#4a5568` on white (≥ 4.5:1) |
| `list` / `listitem` | Serious | Fixture wraps `<list-item-pk>` in `<ul role="list">`; `cart-summary` uses `<span slot="items">` not `<li>` |

## Implementation Notes

**Date:** 2026-03-03
**Outcome:** No component source changes were required. The a11y-audit.html harness was designed with correct ARIA patterns from the start. Scripts in `input-pk`, `textarea-pk`, `progress-bar-pk`, `icon-pk`, `spinner-pk`, `modal-pk` forward aria attributes from the custom element host to the inner semantic elements. Playwright test confirms 0 Critical / 0 Serious violations.

**Residual (backlog):**
- `landmark-unique` (Moderate): `cart-summary` has multiple `<aside>` on the same page in the test harness
- `empty-heading` (Minor): `profile-card-pk` uses `<h2>` populated by `pk-slot`; heading is empty before JS runs (axe catches it post-hydration)
