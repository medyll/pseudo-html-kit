# Story S11-01 — Select: Listbox + Combobox (new Listbox API)

**Epic:** Form Controls Migration  
**Sprint:** Sprint 11  
**Points:** 5  
**Priority:** Must  

---

## User Story

As a **form developer**, I want `select-pk` to use the 2026 Listbox API (`<selectedcontent>`, `appearance: base-select`) with a combobox-style filter option, so that native `<select>` is fully styleable with zero custom dropdown JS.

## Context

Chrome 135+ ships `appearance: base-select` which enables native `<select>` to render as a fully styleable custom listbox. This story adds `select-pk.html` as a new atom and a `combobox-pk.html` molecule with a filter input. Falls back gracefully to a JS-driven dropdown on unsupported browsers.

## Acceptance Criteria

```gherkin
Given select-pk with options
When rendered on Chrome 135+
Then appearance: base-select applies (fully styleable select)
  And ::picker(select) CSS pseudo-element is available

Given select-pk on older browser
When appearance: base-select is not supported
Then fallback styled select renders (no JS error)

Given combobox-pk with options and filter attribute
When user types in filter input
Then displayed options narrow to matching items

Given select-pk with required attribute
When form submitted without selection
Then :invalid pseudo-class applies natively
```

## Technical Notes

- New atom: `src/pseudo-assets/components/atoms/select-pk.html`
- New molecule: `src/pseudo-assets/components/molecules/combobox-pk.html`
- CSS: `@supports selector(::picker(select))` gate for base-select styles
- Fallback: standard `<select>` with minimal custom CSS (no JS dropdown)
- Combobox filter: vanilla JS `oninput` filtering with `hidden` attribute on `<option>`
- Update `src/pseudo-assets/index.js` to export new components
- Add vitest tests in `tests/atoms.client.test.js` and `tests/molecules.client.test.js`

## Out of Scope

- Multi-select listbox
- Async option loading (remote search)
- Virtual scrolling for large option lists
