# Story S11-04 — v0.4.0 GA Release

**Epic:** Release  
**Sprint:** Sprint 11  
**Points:** 3  
**Priority:** Must  

---

## User Story

As a **release manager**, I want a formal v0.4.0 GA release tag pushed to Git with complete release notes, so that users and CI can consume the stable version.

## Context

v0.4.0 scope is fully implemented across Sprints 9, 10, and 11. This story finalizes the release: bumps the version, writes consolidated release notes, and pushes the `v0.4.0` git tag. CI handles npm publish.

## Acceptance Criteria

```gherkin
Given all Sprint 11 stories are done
When release process runs
Then package.json version = "0.4.0"
  And CHANGELOG.md has v0.4.0 entry
  And git tag v0.4.0 is pushed
  And release-notes-v0.4.0.md is complete

Given v0.4.0 tag is pushed
When CI runs
Then npm publish executes automatically (no manual publish)
```

## Technical Notes

- Bump `package.json` version from `0.3.0` → `0.4.0`
- Write `bmad/artifacts/release-notes-v0.4.0.md` consolidating all Sprint 9–11 changes
- Append v0.4.0 entry to `CHANGELOG.md`
- Create `CHANGELOG-v0.4.0.md` summary file
- `git tag v0.4.0 && git push origin v0.4.0`
- **Never run `npm publish` manually — CI only**
