# Story S9-05 — Unit test mocks + updates for new APIs

**Epic:** Testing
**Sprint:** Sprint 09
**Points:** 3
**Priority:** Must

## User Story

As a **developer**, I want the Vitest test suite to properly mock `<dialog>`, Popover API, and ToggleEvent so that migrated components are testable in happy-dom.

## Context

happy-dom does not implement `<dialog>.showModal()`, `HTMLElement.popover`, or `ToggleEvent`. The 4 migrated components need these APIs mocked in `tests/molecules.client.test.js` to validate behavior.

Traces: TST-01, TST-02

## Acceptance Criteria

```gherkin
Given the test suite runs with happy-dom
When modal-pk test executes
Then <dialog> element has showModal() and close() methods available

Given the test suite runs with happy-dom
When dropdown-pk test executes
Then HTMLElement.prototype has showPopover/hidePopover
And ToggleEvent constructor is available

Given all 4 migrated components
When pnpm test runs
Then all existing molecule tests pass (0 regressions)
And new assertions for native API behavior are validated

Given the mock setup
Then it does not pollute other test suites (isolated to molecule tests or beforeEach)
```

## Technical Notes

- Add mock setup in `beforeEach` or top-level setup of `molecules.client.test.js`
- Mock `HTMLDialogElement` with `showModal()`, `close()`, `open` property
- Mock `HTMLElement.prototype.showPopover` / `hidePopover`
- Mock `ToggleEvent` constructor if not available
- Update modal tests: assert `querySelector('dialog.modal')` instead of `querySelector('.modal')`
- Update modal tests: verify `showModal()` called when host `[open]` set
- Update dropdown tests: assert `[popover]` attribute on menu, unique `id`
- Update dropdown tests: verify `aria-expanded` sync via toggle event
- Tooltip tests: unchanged (CSS-only changes, no new JS)
- Notification tests: add test for `[autodismiss]` + dismiss button behavior

## Out of Scope

- E2E tests (S9-06)
- Testing actual browser rendering of anchor positioning or popover

## Dependencies

- S9-01, S9-02, S9-03, S9-04 (component changes must be done first)
