# S12-01 — Carousel: CSS Scroll Snap

Points: 5
Priority: Must

## Summary
Implement `carousel-pk` using CSS Scroll Snap as the primary interaction model. Provide accessible keyboard navigation, focus management, and SSR-friendly markup. Minimize JavaScript; use JS only for progressive enhancements (autoplay, advanced controls).

## Acceptance Criteria
- Component renders with correct layout in modern browsers.
- Keyboard navigation (Left/Right arrows, Home/End) moves focus and scrolls slides.
- ARIA semantics applied: role="region" with accessible name, slides marked appropriately.
- Unit tests verify markup stamping, keyboard handling fallback, and hydration handshake.

## Notes
- Prefer CSS-first approach; fallback for browsers without Scroll Snap will still allow content discovery.
